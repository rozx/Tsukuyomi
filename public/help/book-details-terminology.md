# 📖 术语管理

术语管理功能帮助您维护专有名词、人名、地名等的翻译一致性,确保整本书的翻译质量。

## 💬 什么是术语库

术语库是书籍特定的专有名词翻译对照表,包括:

**常见术语类型**:

- **人名**: 角色名称的翻译
- **地名**: 地点、国家、城市等
- **专有名词**: 组织名称、技能名称、物品名称等
- **特殊概念**: 世界观相关的特定术语
- **文化词汇**: 特定文化背景的词汇

## 🎯 访问术语管理面板

**操作步骤**:

1. 在书籍详情页的左侧边栏
2. 找到"设置菜单"区域
3. 点击"术语设置"按钮
4. 右侧会显示术语管理面板

**面板布局**:

- **顶部工具栏**: 添加术语、搜索、导入导出
- **术语列表**: 显示所有术语的卡片
- **术语详情**: 点击术语查看和编辑详细信息

## ➕ 添加术语

### 手动添加

**操作步骤**:

1. 在术语管理面板
2. 点击"添加术语"按钮(➕ 图标)
3. 在弹出的对话框中填写:
   - **原文**: 原文中的术语(必填)
   - **译文**: 对应的翻译(必填)
   - **类型**: 术语类型(人名/地名/组织/技能/物品/其他)
   - **备注**: 额外的说明信息(可选)
4. 点击"保存"按钮

**填写建议**:

- **原文准确**: 必须与原文完全一致
- **译文统一**: 使用规范的翻译
- **类型明确**: 选择正确的术语类型有助于AI理解
- **备注详细**: 添加必要的说明,如读音、含义等

### 从章节内容添加

**快速添加流程**:

1. 在阅读章节时发现需要添加的术语
2. 选中原文中的术语文本
3. 右键菜单选择"添加为术语"
4. 系统自动填充原文,您只需填写译文
5. 保存即可

**优势**:

- 减少手动输入
- 避免拼写错误
- 保证原文的准确性

### 批量导入

**导入方式**:

1. 准备术语文件(CSV 或 JSON 格式)
2. 在术语面板点击"导入"按钮
3. 选择文件
4. 系统会自动解析并导入术语

**CSV 格式示例**:

```csv
原文,译文,类型,备注
John Smith,约翰·史密斯,人名,主角
Magic Academy,魔法学院,organization,主要场景
Fireball,火球术,skill,
```

**JSON 格式示例**:

```json
[
  {
    "original": "John Smith",
    "translation": "约翰·史密斯",
    "type": "character",
    "note": "主角"
  },
  {
    "original": "Magic Academy",
    "translation": "魔法学院",
    "type": "organization",
    "note": "主要场景"
  }
]
```

## ✏️ 编辑术语

### 修改术语信息

**操作步骤**:

1. 在术语列表中找到要编辑的术语
2. 点击术语卡片
3. 在弹出的对话框中修改信息
4. 点击"保存"按钮

### 修改影响

**术语修改后**:

- 修改不会自动更新已翻译的内容
- 新的翻译会使用更新后的术语
- 建议使用搜索替换功能更新旧译文

**更新已翻译内容**:

1. 修改术语后
2. 使用章节的搜索替换功能
3. 搜索旧译文,替换为新译文
4. 或者重新翻译相关章节

## 🗑️ 删除术语

**操作步骤**:

1. 在术语列表中找到要删除的术语
2. 点击术语卡片旁的"删除"按钮(🗑️ 图标)
3. 在确认对话框中确认删除

**删除影响**:

- 删除后不可恢复
- 已翻译的内容不会自动更改
- AI 翻译时不再使用该术语

## 🏷️ 术语类型

### 人名 (Character)

用于角色名称的翻译。

**特点**:

- 通常需要音译
- 可以添加读音备注
- 建议统一姓名格式(如"姓·名"或"名 姓")

**示例**:

- Harry Potter → 哈利·波特
- Hermione Granger → 赫敏·格兰杰

### 地名 (Location)

用于地点、城市、国家等地理名称。

**特点**:

- 优先使用官方译名
- 虚构地名可以意译
- 保持地名的风格一致

**示例**:

- Hogwarts → 霍格沃茨
- Diagon Alley → 对角巷

### 组织 (Organization)

团体、机构、公司等名称。

**特点**:

- 可以意译体现组织性质
- 保持正式性
- 考虑简称

**示例**:

- Ministry of Magic → 魔法部
- Order of the Phoenix → 凤凰社

### 技能 (Skill)

技能、魔法、招式等名称。

**特点**:

- 通常意译
- 体现技能特点
- 可以保留原文的韵律感

**示例**:

- Fireball → 火球术
- Thunder Strike → 雷霆一击

### 物品 (Item)

道具、装备、物品等名称。

**特点**:

- 根据物品特性命名
- 保持奇幻或科幻感
- 可以使用修饰词

**示例**:

- Elder Wand → 老魔杖
- Philosopher's Stone → 魔法石

### 其他 (Other)

不属于以上类型的术语。

**包括**:

- 世界观概念
- 种族名称
- 货币单位
- 计量单位
- 专有技术名词

## 🔍 术语搜索

### 快速查找术语

**操作步骤**:

1. 在术语面板顶部找到搜索框
2. 输入搜索关键词
3. 列表会实时过滤显示匹配的术语

**搜索范围**:

- 原文
- 译文
- 备注

**搜索技巧**:

- 支持模糊搜索
- 不区分大小写
- 可以搜索部分关键词

### 按类型筛选

**操作步骤**:

1. 在搜索框旁找到类型筛选器
2. 选择要查看的术语类型
3. 列表只显示该类型的术语

**用途**:

- 快速查看特定类型的术语
- 检查某类术语的统一性
- 批量管理同类术语

## 🎨 术语在翻译中的应用

### AI 自动应用

**翻译时**:

- AI 会自动检测原文中的术语
- 使用术语库中的译文替换
- 保持翻译的一致性

**应用优先级**:

1. 完全匹配优先
2. 长术语优先(避免冲突)
3. 最近添加的术语优先

### 术语高亮显示

**在章节内容中**:

- 术语会自动高亮显示
- 鼠标悬停显示术语详细信息
- 点击可以快速跳转到术语管理

**高亮样式**:

- 不同类型的术语使用不同颜色
- 高亮不影响阅读
- 可以在设置中关闭高亮

### 术语使用统计

**查看统计**:

- 在章节工具栏可以看到"使用的术语数量"
- 点击可以查看当前章节使用的术语列表
- 可以看到每个术语的使用次数

**用途**:

- 检查术语是否被正确应用
- 发现遗漏的术语
- 评估章节的复杂度

## 📤 导出和导入

### 导出术语库

**操作步骤**:

1. 在术语面板点击"导出"按钮
2. 选择导出格式 (CSV 或 JSON)
3. 选择保存位置

**用途**:

- 备份术语库
- 在其他项目中使用
- 与他人共享术语库

### 导入术语库

**操作步骤**:

1. 在术语面板点击"导入"按钮
2. 选择术语文件
3. 选择导入模式:
   - **合并**: 保留现有术语,添加新术语
   - **替换**: 清空现有术语,导入新术语
4. 确认导入

**注意事项**:

- 检查文件格式是否正确
- 建议先备份当前术语库
- 大量导入时请耐心等待

## 💡 术语管理最佳实践

### 建立术语库的时机

**推荐流程**:

1. **导入书籍后立即建立**: 先创建基本的术语库
2. **翻译前完善**: 在开始翻译第一章前,尽可能完善术语库
3. **翻译中补充**: 翻译过程中发现新术语及时添加
4. **定期审核**: 定期检查和更新术语库

### 术语命名规范

**统一风格**:

- 人名统一使用"姓·名"或"名 姓"格式
- 地名统一使用官方译名或音译
- 技能名称统一使用"XX术"或"XX之XX"格式

**避免冲突**:

- 一个原文只对应一个译文
- 不同原文不要使用相同译文
- 使用备注区分相似术语

### 术语的优先级

**重要性排序**:

1. **核心角色**: 主角、重要配角的名字
2. **关键地点**: 主要场景、重要地标
3. **核心概念**: 世界观相关的重要术语
4. **常用技能**: 频繁出现的技能或魔法
5. **其他术语**: 次要角色、一般物品等

**建议**:

- 先完善高优先级的术语
- 低优先级术语可以在翻译中逐步添加

### 术语质量控制

**定期审核**:

- 每完成一卷后审核术语库
- 检查术语的一致性
- 修正错误或不当的翻译

**团队协作**:

- 多人翻译时共享术语库
- 统一术语翻译标准
- 定期同步术语库更新

## ❓ 常见问题

### 为什么 AI 翻译时没有使用术语库?

1. 检查术语的原文是否完全匹配
2. 确认术语库已保存
3. 尝试重新翻译该段落
4. 检查 AI 模型是否支持术语库

### 如何处理一词多义的术语?

1. 使用不同的原文表示(如果有)
2. 在备注中说明使用场景
3. 考虑使用章节特殊指令区分
4. 人工翻译时根据上下文选择

### 术语库可以在不同书籍间共享吗?

可以通过导出和导入功能实现:

1. 从一本书导出术语库
2. 在另一本书中导入
3. 选择"合并"模式保留现有术语

### 术语太多会影响性能吗?

- 一般不会影响性能
- 系统会智能缓存常用术语
- 建议控制在1000个术语以内
- 定期清理不再使用的术语

### 如何批量修改术语的类型?

目前需要逐个修改,建议:

1. 导出术语库到 CSV
2. 在 Excel 中批量修改
3. 重新导入(选择"替换"模式)
