# 聊天助手使用指南

## 💬 什么是聊天助手?

**聊天助手 (Chat Assistant)** 是 Tsukuyomi 提供的智能AI对话功能,让您在翻译过程中随时与AI进行深度交流,获得专业的翻译建议、术语解释和风格指导。

每本书籍都有独立的聊天会话,AI会记住您之前的对话内容,确保交流的连贯性和上下文理解。

---

## 🚀 快速开始

### 打开聊天助手

聊天助手在多个页面都可以访问:

1. **从书籍详情页**:
   - 位置: 右上角工具栏
   - 按钮: **💬 聊天** 图标
2. **从章节翻译页**:
   - 位置: 顶部工具栏
   - 按钮: **💬** 图标按钮
3. **从任何书籍相关页面**:
   - 寻找顶部或右上角的 **💬** 聊天图标

点击后,聊天面板会从右侧滑出,占据约 1/3 的屏幕宽度,不会遮挡您正在查看的内容。

### 关闭聊天助手

有多种方式关闭聊天面板:

- **点击关闭按钮**: 面板右上角的 **✕** 图标

**提示**: 关闭面板不会清除聊天历史,下次打开会继续上次的对话。

### 开始对话

在聊天输入框中输入您的问题,按 `Enter` 发送。AI会基于当前书籍的上下文和您的翻译进度给出专业回答。

**快捷键**: `Shift + Enter` 可以在输入框中换行,`Enter` 则发送消息。

## ✨ 核心功能

### 📖 翻译咨询

向AI询问具体段落或词汇的翻译建议:

**示例对话**:

- 💬 "「運命の糸」这个词怎么翻译比较好?"
- 🤖 AI会结合书籍背景提供多种译法,如"命运之线"、"宿命之丝"等,并说明各自的优劣

- 💬 "这段对话的语气应该用正式还是口语化?"
- 🤖 AI会分析角色特点和场景,给出针对性建议

### 🔍 术语解释

不确定某个日语术语的含义?直接问AI:

**示例对话**:

- 💬 "「魔力」和「マナ」有什么区别?"
- 🤖 AI会解释两者在日本轻小说中的常见用法和翻译差异

- 💬 "这本书里的「ギルド」应该译成公会还是行会?"
- 🤖 AI会结合书籍世界观和行业惯例给出建议

### 🎨 风格探讨

询问不同翻译风格的选择:

**示例对话**:

- 💬 "这段战斗描写,直译好还是意译更流畅?"
- 🤖 AI会展示两种风格的示例,分析各自的优势

- 💬 "角色的敬语应该怎么处理?要保留还是转换成中文的语气?"
- 🤖 AI会根据角色性格和场景给出建议

### 📚 上下文记忆

AI会记住您在这本书的聊天历史:

- **连续讨论**: "继续刚才的话题"、"那上个问题呢?" AI都能理解
- **术语一致性**: 如果您之前讨论过某个术语的译法,AI会在后续对话中保持一致
- **剧情理解**: AI会记住您提到的剧情细节,在回答时考虑这些背景

### 🛠️ 实时协作

翻译过程中遇到难题?无需切换页面:

1. 在翻译页面打开聊天助手
2. 描述您遇到的问题
3. AI会基于当前章节内容和已有翻译给出针对性建议
4. 可以直接在对话中讨论多个翻译方案

### ❓ 使用帮助

不知道某个功能怎么用?问AI:

**示例对话**:

- 💬 "如何添加术语表?"
- 🤖 AI会提供详细的操作步骤

- 💬 "怎么切换AI模型?"
- 🤖 AI会引导您完成设置

### 📝 数据管理

**强大的管理功能**: 聊天助手不仅能回答问题,还能直接帮您管理书籍数据,只需用自然语言提出请求即可!

#### 管理角色设定

**示例对话**:

- 💬 "添加一个角色:艾莉丝,金发碧眼的魔法师,性格温柔但战斗时很认真"
- 🤖 AI会自动为您创建角色档案

- 💬 "修改主角的性格设定,他现在变得更成熟了"
- 🤖 AI会更新对应的角色信息

- 💬 "删除配角小明的设定"
- 🤖 AI会确认后删除

#### 管理术语表

**示例对话**:

- 💬 "把「ギルド」添加到术语表,译为'冒险者公会'"
- 🤖 AI会自动添加术语条目

- 💬 "查看所有术语"
- 🤖 AI会列出当前书籍的所有术语

- 💬 "将「魔力」的译法改为'魔法力'"
- 🤖 AI会更新术语

#### 管理记忆

**示例对话**:

- 💬 "记住:这本书的世界观是蒸汽朋克+魔法"
- 🤖 AI会创建记忆条目供后续参考

- 💬 "记录:主角在第5章获得了传说之剑"
- 🤖 AI会保存剧情要点

- 💬 "查看所有记忆"
- 🤖 AI会展示已保存的记忆

#### 管理书籍详情

**示例对话**:

- 💬 "修改书名为《异世界冒险记》"
- 🤖 AI会更新书籍信息

- 💬 "添加作者:山田太郎"
- 🤖 AI会补充元数据

- 💬 "更新书籍简介"
- 🤖 AI会引导您输入新的简介

**优势**: 无需在多个页面间切换,边翻译边管理,一切尽在对话中完成!

## 💡 最佳实践

### 1. 简洁提问即可

**系统会自动附加上下文**: Tsukuyomi 会自动将当前书籍的元数据、术语表、角色设定等信息附加到您的问题中,所以您不需要重复提供这些背景。

**示例**:

- ✅ **推荐**: "「必殺技」怎么翻译?"
- ⚠️ **不必要**: "在《XXX》这本书的第3章战斗场景中,「必殺技」这个词应该翻译成什么?角色是个中二病高中生。"

AI已经知道您正在翻译哪本书、当前章节是什么,以及已定义的角色设定,因此简洁提问反而更高效!

### 2. 利用会话历史

如果您之前讨论过某个术语或风格问题,可以直接引用:

- "还是用之前说的那个译法吧"
- "按照我们讨论的角色设定来翻译"

### 3. 请求多种方案

不确定哪个译法更好?让AI提供选项:

- "请给出3种不同风格的翻译"
- "这段话的直译和意译分别是什么样?"

### 4. 讨论译文改进

对AI生成的翻译不满意?在聊天中具体说明:

- "这段翻译感觉太生硬了,能更口语化一些吗?"
- "角色的语气应该更傲娇一点"

### 5. 建立术语库

在对话中确定了某个术语的标准译法后,可以请AI帮您记录:

- "把「魔力」→「魔力」这个对应加入术语表"
- AI会引导您完成添加(或在未来版本中自动添加)

## 🎯 应用场景

### 场景 1: 新手译者入门

**情况**: 刚开始翻译轻小说,不确定很多术语的标准译法

**使用方式**:

1. 遇到不确定的术语就向AI询问
2. 讨论不同译法的优劣
3. 建立自己的术语表

### 场景 2: 专业译者润色

**情况**: 翻译经验丰富,但想进一步优化译文质量

**使用方式**:

1. 将翻译好的段落贴给AI审阅
2. 讨论语言风格和文学性
3. 探讨更地道的中文表达

### 场景 3: 难点攻坚

**情况**: 遇到复杂的长难句或文化梗

**使用方式**:

1. 贴出原文和自己的初步翻译
2. 向AI请教句子结构和深层含义
3. 讨论如何在中文中重现原文效果

**使用方式**:

1. 在聊天中与AI讨论并确定翻译规范
2. 将这些规范应用到所有章节
3. 用AI检查不同章节的风格是否统一

---

## ⚠️ 注意事项

1. **AI建议仅供参考**: AI的建议基于大量语料,但最终翻译决策应由您来做
2. **保护隐私**: 如果书籍涉及未公开内容,请注意不要在对话中透露敏感信息
3. **API用量**: 每次对话都会消耗API额度,建议合理使用
4. **会话限制**: 非常长的会话可能影响AI的回答质量,必要时可以开启新会话
