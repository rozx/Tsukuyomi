## ADDED Requirements

### Requirement: 触控等价操作路径
系统 MUST 为所有仅鼠标友好的关键交互提供触控等价路径，且结果与桌面交互一致。

#### Scenario: 章节排序的触控等价
- **WHEN** 用户在触控设备上需要调整章节顺序
- **THEN** 系统 MUST 提供可拖拽手柄或上移下移按钮以完成排序

#### Scenario: 小屏端快速操作可见
- **WHEN** 用户在手机端执行段落操作、翻译提交或面板切换
- **THEN** 系统 MUST 提供可见按钮入口，不依赖悬停或右键菜单

### Requirement: 触控命中区与反馈标准
系统 MUST 保证可交互控件满足移动端命中区要求并提供即时反馈，降低误触率。

#### Scenario: 可点控件命中区合规
- **WHEN** 用户在手机端点击工具栏按钮或列表操作项
- **THEN** 系统 MUST 使触控命中区不小于最小可用标准并提供按压反馈

### Requirement: 手势与滚动冲突可控
系统 MUST 管理拖拽、横向滑动与纵向滚动冲突，确保主流程操作稳定。

#### Scenario: 列表滚动不触发误拖拽
- **WHEN** 用户在可排序列表中执行纵向滚动
- **THEN** 系统 MUST 默认识别为滚动，且仅在明确拖拽手势下触发排序

#### Scenario: 侧滑面板与页面滚动不冲突
- **WHEN** 用户在手机端打开侧滑面板后进行内容滚动
- **THEN** 系统 MUST 将滚动焦点限制在当前活动容器，避免背景页面误滚动
